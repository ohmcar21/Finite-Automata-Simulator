Foundation

What is Automata Theory?

Automata Theory is a branch of theoretical computer science that studies abstract computing devices called automata. It forms the mathematical foundation for understanding what computers can and cannot do, and is essential for compiler design, pattern matching, and formal language processing.

Finite Automaton

A mathematical model of computation with a finite number of states. It reads an input string symbol by symbol and either accepts or rejects it.

States & Transitions

States represent configurations of the machine. Transitions are rules that move the machine from one state to another based on the input symbol being read.

Alphabet & Input

The alphabet (Σ) is a finite set of symbols the automaton can read. An input string is a sequence of symbols from this alphabet that the automaton processes.

Start State

The start state (q₀) is the initial state where the automaton begins processing the input. Every FA has exactly one start state, usually indicated by an incoming arrow.

Accept (Final) State

An accept state (or final state) means the input is recognized by the automaton. If processing ends in an accept state, the string is accepted; otherwise it is rejected.

Transition Function (δ)

The transition function δ defines the rules: given a current state and input symbol, it determines the next state. It is the "program" that drives the automaton.

Formal Definition

A Finite Automaton is a 5-tuple

M = ( Q, Σ, δ, q₀, F )
  • Q — Finite set of states
  • Σ — Finite alphabet (input symbols)
  • δ — Transition function
  • q₀ — Start state (q₀ ∈ Q)
  • F — Set of accept/final states (F ⊆ Q)
Type 1

DFA — Deterministic Finite Automaton

A DFA processes input one symbol at a time, making exactly one move at each step. There is no ambiguity — the next state is always uniquely determined.

Rules & Restrictions

  • ✓ Exactly one transition per symbol from every state
  • ✗ No ε (epsilon) transitions allowed
  • ✓ Behavior is fully deterministic — one unique path
  • ✓ Every state must have a transition for every symbol in Σ
  • ✓ Transition function: δ : Q × Σ → Q

Type 2

NFA — Non-Deterministic Finite Automaton

An NFA can be in multiple states simultaneously. It explores all possible paths and accepts the input if any path leads to an accept state.

Rules & Characteristics

  • ✓ Multiple transitions allowed for the same symbol
  • ✓ ε (epsilon) transitions allowed — move without reading input
  • ✓ Behavior is non-deterministic — multiple paths explored
  • ✓ States may have zero transitions for some symbols
  • ✓ Transition function: δ : Q × (Σ ∪ {ε}) → P(Q)

Side by Side

DFA vs NFA Comparison

Property DFA NFA
Transitions per symbol Exactly 1 0, 1, or many
ε-transitions Not allowed Allowed
Determinism Deterministic Non-deterministic
Transition function δ : Q × Σ → Q δ : Q × (Σ ∪ {ε}) → P(Q)
Ease of design Harder Easier
Implementation Simple Requires backtracking / subset construction
Expressive power Equivalent — both recognize regular languages
